App Name: Social Sage
(open source version)
Purpose: Create one post → automatically send it to 13 social media platforms in the correct way. Safe, easy, automatic.
The app takes your one post (title + Description + hashtags + photo/video) and sends it to each platform in the correct way. No extra questions to the user. Everything happens automatically.

Here is the full list of technical requirements for Social Sage.
1. Allover data save on user device (open source version)
Used for:
Storing user login tokens (from all 13 platforms)
Saving scheduled posts
Keeping user library (photos, videos, music files)
Storing analytics data (views, likes, comments)
Running scheduled posts automatically
Sending push notifications when a post goes live
Why best: No server management, all on user device

2. Official APIs from Each Platform
Needed for: Actually posting content (text, photo, video, story, etc.) to each platform
Examples:
Facebook Graph API
Instagram Graph API
Twitter/X API
TikTok API
YouTube Data API
LinkedIn API
Pinterest API
Telegram Bot API
WhatsApp Business API (for status/channels – limited)
Snapchat API
Reddit API
Discord Bot API
WeChat API
Important: We must get developer account + API keys for each one

3. OAuth2 Authentication
Needed for: Safe one-time login for all 13 platforms
How it works: User logs in once → we get access token → token saved on user device

4. Security Rules
Needed for: Protect user data — only the owner can see their own tokens, library, posts

5. Push Notification
Needed for: Send notification to user when scheduled post is published
(example: "Your post to Instagram is live!")

6. API Limitations
Some platforms may have limited or no public APIs (e.g., WhatsApp, Snapchat, WeChat). If posting or analytics is not possible for a platform, the app should show a message to the user and disable unsupported features for that platform.

7. Music Copyright
Only allow user-uploaded music for posts. Do not fetch or use copyrighted or trending music from external sources to avoid copyright issues.

8. Replying to Messages
Not all platforms allow replying via API. If direct reply is not supported, show an error message: "Not allowed: direct reply is not supported for this platform."

9. Data Encryption & Security
- Encrypt all sensitive data (tokens, user info, media) stored on the device.
- Use strong encryption (e.g., AES-256).
- Store encryption keys securely (use OS-provided secure storage, e.g., Android Keystore, iOS Keychain).
- Biometric/Password Protection: Require fingerprint or password to open the app.
- Use OS-level biometric APIs for authentication.
- Auto-lock app after phone lock.
